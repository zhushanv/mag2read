# Mag2Read 功能亮点文档

文档版本：V1.0  
适用项目：Mag2Read —— 智能杂志文章电子化与导读系统  
撰写日期：2026-06-27

## 1. 文档目的

本文档用于整理 Mag2Read 项目在原有需求基础上可以进一步强化的功能亮点，重点突出项目在课程设计中的展示价值。与基础需求文档相比，本文更关注“为什么这个功能有亮点、如何实现、答辩时如何讲清楚”。

Mag2Read 的核心定位不是普通 OCR 工具，也不是简单的 PDF 摘要工具，而是面向扫描版杂志、期刊长文的电子书化清洗与 AI 导读系统。系统应重点体现以下完整链路：

```text
扫描 PDF / 图片上传
        ↓
版面分析与 OCR 识别
        ↓
阅读顺序恢复
        ↓
页眉页脚与噪音清洗
        ↓
正文结构重建
        ↓
EPUB / Markdown 导出
        ↓
AI 导读与阅读卡片生成
```

## 3. 杂志双栏阅读顺序恢复

### 3.1 功能说明

扫描版杂志和期刊常见双栏、三栏或图文混排。普通 OCR 往往只能给出文本块和坐标，如果直接按从上到下、从左到右拼接，容易出现左右栏交错的问题。阅读顺序恢复功能的目标是根据 OCR 坐标、文本块位置和版面结构，重新生成符合人类阅读习惯的正文顺序。

### 3.2 实现思路

建议设计 `ReadingOrderResolver` 模块，处理流程如下：

```text
页面版心检测 → 栏数判断 → 文本块分类 → 栏内排序 → 跨栏合并 → 跨页拼接
```

关键规则包括：

- 根据文本块横坐标分布判断页面是单栏还是双栏。
- 对同一栏内的文本块按照纵坐标排序。
- 对标题、正文、图注设置不同优先级，避免标题被错误插入正文中间。
- 对页尾未结束的句子与下一页页首文本进行跨页拼接。

### 3.3 答辩表达

可以将该功能作为项目的核心算法点来讲。重点说明系统不是简单调用 OCR，而是在 OCR 结果之上做了版面理解和文本重建，使扫描杂志真正变成适合阅读的连续文章。

## 4. 页眉页脚指纹去噪

### 4.1 功能说明

杂志和期刊中常见页眉、页脚、页码、栏目名、版权信息等重复内容。如果这些内容被直接保留，会干扰正文阅读，也会影响后续 AI 摘要结果。页眉页脚指纹去噪功能用于自动识别并过滤这些重复噪音。

### 4.2 实现思路

建议结合坐标规则、文本重复率和正则表达式进行综合判断：

```text
noise_score = 坐标边缘得分 + 跨页重复得分 + 短文本得分 + 页码格式得分
```

具体规则示例：

- 位于页面顶部 10% 或底部 10% 的文本块，提高噪音得分。
- 多页反复出现的短文本，判定为页眉或页脚。
- 纯数字、罗马数字、`- 12 -`、`第 12 页` 等格式判定为页码候选。
- 与正文相比字号明显较小、长度较短的文本块，优先进入噪音候选池。

### 4.3 答辩表达

该功能适合包装为“排版噪音过滤算法”。答辩时可以展示过滤前后的文本对比，并说明噪音识别不是只靠一个规则，而是多特征打分。

## 5. 可解释清洗预览

### 5.1 功能说明

为了避免系统处理过程过于黑盒，前端可以提供清洗预览页面。用户在左侧查看原始页面图片和 OCR 框，在右侧查看清洗后的正文内容。不同类型的文本块用不同颜色标记。

### 5.2 展示内容

建议区分以下区块类型：

| 区块类型 | 显示颜色建议 | 说明 |
|---|---|---|
| 标题 | 蓝色 | 文章标题、小标题 |
| 正文 | 绿色 | 保留并参与导出的正文 |
| 图注 | 紫色 | 图片、表格说明文字 |
| 页眉页脚 | 橙色 | 被系统识别为重复噪音 |
| 页码 | 灰色 | 被过滤的页码 |
| 疑似异常 | 红色 | 需要人工确认的文本块 |

用户点击被删除的区块时，可以显示删除原因，例如：

```text
删除原因：该文本位于页面底部区域，且连续 6 页重复出现，系统判定为页脚。
```

### 5.3 答辩表达

该功能可以明显增强演示效果。老师能够直观看到 OCR 结果、清洗规则和最终正文之间的关系，避免项目看起来只是普通文件转换工具。

## 6. 转换质量评分

### 6.1 功能说明

系统在完成 OCR 与清洗后，为每个任务生成一个转换质量评分，帮助用户判断本次转换结果是否可靠。评分不要求绝对准确，重点是通过多个指标反映处理质量。

### 6.2 评分指标

| 指标 | 含义 | 示例 |
|---|---|---|
| OCR 平均置信度 | 识别文字是否可靠 | 平均置信度 0.91 |
| 噪音过滤率 | 页眉页脚、页码等内容占比 | 过滤 8.5% 噪音文本 |
| 段落连贯度 | 是否存在大量碎片化短行 | 连贯度 82/100 |
| 阅读顺序异常数 | 是否存在疑似跨栏错序 | 发现 2 处疑似异常 |
| 图表占比 | 判断是否需要人工复核 | 图表区域占 18% |

### 6.3 输出示例

```text
本次转换质量：86/100
OCR 识别整体可靠。第 5 页和第 9 页存在图文混排，建议在导出前进行预览确认。
```

### 6.4 答辩表达

质量评分可以体现系统工程化思维。它不仅完成转换，还能对结果进行自我评估，便于用户判断是否需要人工校正。

## 7. AI 阅读卡片

### 7.1 功能说明

原需求中的 AI 导读可以从“200 字摘要 + 关键词”升级为结构化阅读卡片。这样既能体现 AI 能力，也更符合学生阅读长文时的实际需求。

### 7.2 阅读卡片内容

建议生成以下内容：

- 一句话概括：用一句话说明文章主要讨论什么。
- 核心摘要：约 200 字，概括文章主要观点。
- 关键词：提取 3 到 5 个关键词。
- 文章结构：按段落或小标题生成文章脉络。
- 精读建议：指出最值得重点阅读的段落。
- 快速浏览版：提炼 3 个要点。
- 思考问题：生成适合课堂汇报或读书笔记的问题。

### 7.3 输出示例

```text
阅读卡片

一句话概括：本文主要讨论人工智能工具如何改变长文阅读和知识整理方式。

核心摘要：……

关键词：OCR、版面分析、电子书转换、AI 导读

推荐精读段落：第 3 段、第 5 段、第 8 段

思考问题：
1. 扫描文档清洗为什么会影响 AI 摘要质量？
2. 双栏阅读顺序恢复和普通 OCR 有什么区别？
```

### 7.4 答辩表达

可以强调系统先清洗正文，再生成导读。与直接把原始 PDF 丢给大模型相比，这种流程能减少页眉页脚、错序文本对摘要质量的干扰。

## 8. 图表导读与缩略图保留

### 8.1 功能说明

杂志文章中经常包含图片、表格和图注。如果导出时直接删除图片，会影响阅读完整性；如果完整保留大图，又可能影响 EPUB 体积和阅读体验。因此建议采用“缩略图 + 图表导读”的方式处理。

### 8.2 实现思路

- 通过版面分析识别图片和表格区域。
- 将原图区域裁切为缩略图，保存到任务目录。
- 对图注和图片附近文本进行合并分析。
- 可选调用多模态模型，为图片生成简短说明。
- 在 Markdown 或 EPUB 中插入图片缩略图和图表导读。

输出示例：

```markdown
![图 2 缩略图](images/figure_02.png)

图表导读：该图主要展示了不同页面清洗策略对 OCR 结果的影响，其中双栏重排后的文本连贯性最好。
```

### 8.3 答辩表达

该功能可作为项目加分项。它体现系统不仅处理文字，还能兼顾图文混排文档的完整阅读体验。

## 9. 电子阅读器适配模式

### 9.1 功能说明

不同阅读设备对格式有不同要求。系统可以在导出时提供多种模式，使 EPUB 或 Markdown 更适合实际阅读。

### 9.2 模式设计

| 模式 | 适用场景 | 处理策略 |
|---|---|---|
| Kindle 模式 | 电子墨水屏阅读 | 简化样式、压缩图片、减少复杂表格 |
| 手机模式 | 手机浏览器或阅读 App | 增大段距，保留适量图片 |
| 学术整理模式 | 写读书笔记或课程汇报 | 保留标题层级、图表编号和引用信息 |
| 纯文本模式 | 导入笔记软件 | 只保留正文和基础标题 |

### 9.3 答辩表达

该功能能强化项目的实用价值。系统不是简单导出文件，而是根据阅读场景生成不同版本。

## 10. 本地隐私模式

### 10.1 功能说明

用户可能会处理教材扫描件、内部资料或未公开论文，因此系统应支持本地隐私模式。在该模式下，OCR 和清洗流程在本地完成，AI 导读可选择关闭或由用户自行配置 API Key。

### 10.2 功能策略

- 默认在本地保存上传文件和中间结果。
- 支持关闭云端 AI 调用，只生成 OCR 和 EPUB。
- API Key 不写入代码仓库，通过环境变量或配置文件读取。
- 任务完成后可一键清理上传文件、页面图片和中间 JSON。

### 10.3 答辩表达

可以将该功能表述为“本地清洗 + 可选云端 AI”的混合架构，在成本、隐私和效果之间保持平衡。

## 11. 人工校正闭环

### 11.1 功能说明

OCR 结果不可能完全准确，因此系统可以提供轻量级人工校正功能。用户在预览页面中修改识别文本、恢复误删区块或删除误保留噪音，随后重新生成 EPUB 和 Markdown。

### 11.2 最小可行版本

建议先实现以下能力：

- 在右侧正文预览中直接编辑段落文本。
- 对被过滤的区块提供“恢复到正文”按钮。
- 对正文区块提供“标记为噪音”按钮。
- 保存用户修正后的文本，重新执行导出流程。

### 11.3 答辩表达

人工校正闭环可以让系统从一次性转换工具变成可用的文档处理平台。即使 OCR 出现误差，用户也有办法修复并导出最终版本。

## 12. 建议开发优先级

### 12.1 第一阶段：保证核心闭环

优先完成以下功能，确保项目能够完整演示：

1. 文件上传与任务创建。
2. PDF 页面渲染。
3. OCR 识别与坐标 JSON 输出。
4. 双栏阅读顺序恢复。
5. 页眉页脚和页码过滤。
6. Markdown / EPUB 导出。

### 12.2 第二阶段：强化演示效果

在核心闭环稳定后，增加以下功能：

1. 清洗前后对比预览。
2. OCR 区块高亮。
3. 删除原因展示。
4. 转换质量评分。
5. AI 阅读卡片。

### 12.3 第三阶段：作为加分项扩展

如果时间充足，再考虑：

1. 图表导读与缩略图保留。
2. 多种电子阅读器适配模式。
3. 本地隐私模式。
4. 人工校正闭环。
5. 批量处理和历史记录管理。

## 13. 答辩展示建议

答辩时建议围绕“原始扫描件难读，系统如何一步步变得好读”展开，而不是只展示上传和下载。

推荐演示流程：

1. 打开一份双栏扫描杂志 PDF，展示在小屏设备上阅读困难。
2. 上传文件，展示任务处理流水线。
3. 展示 OCR 区块和版面分类结果。
4. 展示页眉页脚、页码等噪音如何被识别和过滤。
5. 展示阅读顺序恢复后的正文。
6. 展示 AI 阅读卡片。
7. 下载 EPUB 或 Markdown，并展示最终阅读效果。

答辩时可以重点强调两个创新点：

- 排版噪音过滤算法：通过坐标规则、重复文本检测和正则识别过滤页眉页脚。
- AI 导读增强阅读：在清洗后的正文基础上生成摘要、关键词、结构大纲和精读建议。

## 14. 总结

Mag2Read 的亮点应集中在“扫描杂志电子书化”这一垂直场景上。它与普通 OCR 工具的区别在于，不只识别文字，还要恢复阅读顺序、清除排版噪音、重建正文结构；它与普通 AI PDF 阅读工具的区别在于，先把扫描件清洗成高质量文本，再进行 AI 导读。

因此，项目最适合形成以下技术主线：

```text
OCR 识别是基础，版面清洗是核心，电子书导出是结果，AI 导读是增强。
```
